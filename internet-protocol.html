Internet Protocol

A very important protocol is the internet protocol, also known as IP. There are 2 versions of internet protocol: one for IPv4-addresses named IPv4, and one for IPv6-addresses named IPv6. IP serves as a foundation for many other protocols which are built on top of IP.

IP packets contain two main parts: a header and a body. The header contains all of the metadata, and the body contains the actual information being sent in the packet. (Metadata is the data that describes the qualities of the packet, such as the length of the packet, the IP address to forward the packet to, etc.). The header of the packet is different for both versions of IP, so we will pick them apart separately.

In order for computers to communicate, they each have a unique address known as an IP address. There are 2 types of IP addresses: IPv4 and IPv6. The reason we have two types is because when the internet was created, the creators didn't expect a lot of people to use it. There are only a limited amount of IPv4 addresses, and we are slowly running out of them. Therefore, a new format named IPv6 that we are slowly trying to transition to was created. While IPv4 uses 32 bits, an IPv6 address uses 128 bits. Furthermore, there are differences in how they work and what the different parts mean.

The IPv4 header

The IPv4 header is generally split into 5 or 6 words (a word being 32 bits). Here are the parts of the IPv4 header:

The following table describes the function of each of these parts of the IPv4 header:

Version For IPv4 addresses it is always the number 4 (0100)
IHL Total header length in 32-bit words. The maximum value is 15.
DS (differentiated services) Used to mark packets to be treated with priority by routers. Together with ECN, this replaced the old 8-bit Type of Service field
ECN (explicit congestion notification) Routers experiencing congestion can mark these bits to tell the sender that the transmission rate should be reduced
Total length Specifies packet size before fragmentation.
Identification A unique number used to identify fragmented packets that were originally one
Flags Specifies some flags as to if and how the packet was fragmented
Fragment offset Specifies which fragment of a packet this packet is
Time To Live (TTL) A number (usually 64) that is decremented each time a router receives the packet to avoid forwarding loops. Once at 0, the packet gets discarded.
Protocol Describes the protocol used in the body of the IP address
Header Checksum Used to check a packet for corruption (if it is corrupted, it can be discarded)
Source address IPv4 address of the source of the packet
Destination address IPv4 address of the destination of the packet
IP options Different options for controlling the packet
Padding Padding bits for alignment

The IPv6 header

A lot of these fields work similarly to the IPv4 fields. A lot of fields have been removed because they were unnecessary and IPv6 was designed with simplicity in mind. The following table describes the function of some of the new parts in the IPv6 header:

Version For IPv6, it is set to 6 (0110)
Traffic class Used to specify priority when routing the packets
Source/destination address Same as IPv4, but this time as IPv6 addresses
Flow label Used to identify sequences of packets travelling to some shared destination
Payload length Specifies the length of the payload
Next header Specifies internet-layer options
